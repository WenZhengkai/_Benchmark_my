## Module name
dut

### Overview
The dut module is the instruction decoding unit in the RISC-V processor, responsible for decoding instructions from the IFU (Instruction Fetch Unit), generating control signals and data source selection signals, and passing the decoded information to the ISU (Issue Unit).

The main functions include:
1. Instruction type identification and classification
2. Generate execution unit control signals
3. Immediate expansion
4. Register file read and write control
5. Data source selection

### Parameters
The module inherits from `NPCModule` and uses the following parameters:
- `XLen`: data width (usually 32 or 64)
- `IndependentBru`: Boolean value indicating whether to use an independent branch unit

### Internal logic
1. **Handshake processing**:
- Use `HandShakeDeal` to process the ready/valid handshake signals from IFU and to ISU
- `AnyInvalidCondition` is used to mark invalid conditions, which is set to false in this module
- Directly call the function: HandShakeDeal(io.from_ifu, io.to_isu, AnyInvalidCondition)

2. **Instruction decoding**:
- Use `ListLookup` to look up the instruction decoding table (`DecodeTable`) to get the control signal
- The decoding information includes:
- Instruction type (`instType`)
- Functional unit type (`fuType`)
- Operation type (`fuOpType`)
- Source operand 1 type (`fuSrc1Type`)
- Source operand 2 type (`fuSrc2Type`)
val instType :: fuType :: fuOpType :: fuSrc1Type :: fuSrc2Type :: Nil = ListLookup(inst, Instructions.DecodeDefault, Instructions.DecodeTable)

3. **Control signal generation**:
Refer to val ctrl = new CtrlSignal in IO port
- Register write enable (`rfWen`): Determined by the instruction type, call the `isRegWrite` function above
- Extract the source register in the instruction (`rs1`, `rs2`) and destination register (`rd`) fields
- Memory write enable (`MemWrite`): Set for S type instructions, false.B at other conditions
- Result source select (`ResSrc`): Selector output,
   - Load instruction (inst(6,0) === "b0000011".U) set to 1.U
   - CSR operation (inst(6,0) === "b1110011".U) set to 2.U
   - Default is 0.U
- Source operand type (`fuSrc1Type`, `fuSrc2Type`), functional unit type (`fuType`), operation type (`fuOpType`) are assigned by the instruction decoding result
4. **Immediate extension**:
Use Cat() for splicing
Use SignExt(a:UInt, len:Int) for signed extension, where len represents the bit width of the final result
- Different immediate extensions are performed to XLen bits according to the instruction type, and the extension is performed according to the RISCV32I instruction set specification:
- I type: 12-bit sign extension
- U type: 20-bit immediate value is shifted left 12 bits and extended
- J type: 20-bit sign extension (joined in five parts), pay attention to the order and fill the lowest bit with 0
- S type: 12-bit sign extension (joined in two parts)
- B type: 12-bit sign extension (joined in five parts), pay attention to the order and fill the lowest bit with 0

5. **Data source preparation**:
Refer to the port signal bundle `DataSrc` above
- Other data sources are currently assigned as DontCare at once, data <> DontCare
- Store the immediate extension result in `data.imm` 

6. **Output connection**
connnect all signal bundles of io.to_isu(CtrlFlow, CtrlSignal, DataSrc)
