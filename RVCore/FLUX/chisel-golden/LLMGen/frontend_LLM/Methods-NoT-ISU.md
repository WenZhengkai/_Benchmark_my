## Module - ISU (Issue Unit)

## Overview
This module implements an Instruction Issue Unit (ISU) with a scoreboard mechanism. The main functions include:
1. Track register usage status through the scoreboard to detect data hazards
2. Control the flow of instructions from the decoding stage (IDU) to the execution stage (EXU)
3. Handle operand forwarding and immediate number selection
4. Use the handshake protocol to manage pipeline control

## Parameters
- `maxScore` (ScoreBoard parameter): The maximum count value of each register in the scoreboard, which determines the depth of data hazard detection
- `NR_GPR` (from HasNPCParameter): The number of general registers
- `XLen` (from HasNPCParameter): Data bit width

## Input/output interface
val io = IO(new Bundle{
val from_idu = Flipped(Decoupled(new DecodeIO)) // Instruction information from the decoding unit, introduced later
val to_exu = Decoupled(new DecodeIO) // Instruction information sent to the execution unit
val wb = Input(new WbuToRegIO) // Write back unit feedback, Regarding the busy state of general registers, it will be introduced later.
val from_reg = new Bundle { // Register value read from register file
val rfSrc1 = Input(UInt(XLen.W))
val rfSrc2 = Input(UInt(XLen.W))
}
})

class WbuToRegIO extends NPCBundle { // Signal bundle for WBU to write data back to Regfile, which is used as the basis for updating the busy state of registers
val rd = UInt(5.W)
val Res = UInt(XLen.W)
val RegWrite = Bool()
}

class DecodeIO extends NPCBundle {

val cf = new CtrlFlow // Control flow information (original information from IFU)
val ctrl = new CtrlSignal // Control signal, the signal bundle will be introduced below
val data = new DataSrc // Data source information, the signal bundle will be introduced below

}

// The following Bundle has been defined and does not need to be written repeatedly
class CtrlFlow extends NPCBundle { // Signal bundle describing the instruction characteristics of the current stage
val inst = UInt(32.W) // Current control flow instruction
val pc = UInt(XLen.W) // Instruction address
val next_pc = UInt(XLen.W) // Next instruction address, generated by branch predictor, may not be correct before being checked and corrected by the backend
val isBranch = Bool() // Is it a branch instruction
}
class CtrlSignal extends NPCBundle { // Control signal bundle
val MemWrite = Bool()
val ResSrc = UInt()
val fuSrc1Type = FuSrcType()
val fuSrc2Type = FuSrcType()
val fuType = FuType()
val fuOpType = FuOpType()
val rs1 = UInt(5.W)
val rs2 = UInt(5.W)
val rfWen = Bool()
val rd = UInt(5.W)

}

class DataSrc extends NPCBundle {
val fuSrc1 = UInt(XLen.W) // Functional unit operand
val fuSrc2 = UInt(XLen.W)
val imm = UInt(XLen.W) // Immediate value

val Alu0Res = Decoupled(UInt(XLen.W)) // ALU calculation result
val data_from_mem = UInt(XLen.W) // Data read from memory
val csrRdata = UInt(XLen.W) // Data read from CSR
val rfSrc1 = UInt(XLen.W) // Data read from register
val rfSrc2 = UInt(XLen.W)

}

## Internal logic



**ISU module main logic:**
1. **Data hazard detection**:
- Instantiate ScoreBoard
- Check whether the two source registers are occupied through the isBusy function of ScoreBoard
- Block instruction issuance when an hazard is detected, AnyInvalidCondition is set to valid, and the function HandShakeDeal(io.from_idu, io.to_exu, AnyInvalidCondition) Handle handshake signal

2. **regfile connection function**:

- Define def rs1_rs2(rfSrc1: UInt, rfSrc2: UInt): (UInt, UInt)

Directly assign parameter rfSrc1/rfSrc2 to to_exu port signal in the function

Use rs1/rs2 as function output

2. **Operand processing**:

Reference port to_exu.bits.

- Get operand value from register file

- Select operand source according to `fuSrc1Type/fuSrc2Type`:

- Register value, PC, immediate value or fixed value (0/4), the following is the mapping

FuSrcType.rfSrc1 -> rfSrc1,

FuSrcType.pc -> io.from_idu.bits.cf.pc,

FuSrcType.zero -> 0.U

FuSrcType.rfSrc2 -> rfSrc2,
FuSrcType.imm -> io.from_idu.bits.data.imm,
FuSrcType.four -> 4.U

4. **Scoreboard update**:
create an object of ScoreBoard, val sb = new ScoreBoard(3)
- When the instruction is issued (setMask): mark the target register as occupied
- When the write back is completed (clearMask): release the target register
- Use the mask mechanism to efficiently update the status of multiple registers
val wbuClearMask = Mux(io.wb.RegWrite, sb.mask(io.wb.rd), 0.U)
val isFireSetMask = Mux(inBits.ctrl.rfWen && out.fire, sb.mask(inBits.ctrl.rd), 0.U)
sb.update(isFireSetMask, wbuClearMask)
5. **Connect Output**:
- Pass all DecodeIO Bundle from the decoding stage to the execution stage after processing
- Some signals in DataSrc generated in this module should be overrided connect to exu port

## Design Task
### Task 1: **Data Hazard Detection**
**Objective:** Implement data hazard detection using the ScoreBoard mechanism and handle handshake signals appropriately.
**Steps:**
1. Instantiate a `ScoreBoard` using the `maxScore` parameter.
2. Connect `ScoreBoard` logic to check if source registers (`rs1`, `rs2`) are busy using `isBusy` function.
3. Manage blocking of instruction issuance if a hazard is detected:
   - Define `AnyInvalidCondition`.
   - Call `HandShakeDeal(io.from_idu, io.to_exu, AnyInvalidCondition)` to control the handshake protocol between IDU and EXU.


---

### Task 2: **Regfile Connection Function**
**Objective:** Define a function to connect register file outputs (`rfSrc1`, `rfSrc2`) to the EXU.
**Steps:**
1. Define a function:
   ```scala

    def rs1_rs2(rfSrc1 : UInt, rfSrc2 : UInt): (UInt, UInt) = {
        this.io.from_reg.rfSrc1:= rfSrc1
        this.io.from_reg.rfSrc2:= rfSrc2
        (this.outBits.ctrl.rs1 , this.outBits.ctrl.rs2)
    }
   ```

---

### Task 3: **Operand Processing**
**Objective:** Process operands based on the instruction's functional unit (FU) source type.
**Steps:**
1. Reference the `fuSrc1Type` and `fuSrc2Type` in `io.from_idu.bits.ctrl`.
2. Select and assign source operand values:
   - For `fuSrc1`, map:
     - `FuSrcType.rfSrc1` → `rfSrc1`
     - `FuSrcType.pc` → `io.from_idu.bits.cf.pc`
     - `FuSrcType.zero` → `0.U`
   - For `fuSrc2`, map:
     - `FuSrcType.rfSrc2` → `rfSrc2`
     - `FuSrcType.imm` → `io.from_idu.bits.data.imm`
     - `FuSrcType.four` → `4.U`
3. Update operand values in `io.to_exu.bits`.

---

### Task 4: **Scoreboard Update Logic**
**Objective:** Maintain register occupancy status using `ScoreBoard`’s masking mechanism.
**Steps:**
1. Define a `ScoreBoard` object:
   ```scala
   val sb = new ScoreBoard(maxScore)
   ```
2. Update and use masks for tracking status:
   - On instruction issue (`setMask`):
     - If `io.to_exu.fire` and `ctrl.rfWen` are valid, set the mask for `ctrl.rd`:
       ```scala
       val isFireSetMask = Mux(inBits.ctrl.rfWen && io.to_exu.fire, sb.mask(inBits.ctrl.rd), 0.U)
       ```
   - On instruction writeback (`clearMask`):
     - Use `io.wb` signals to release the mask for `wb.rd` if `wb.RegWrite` is true:
       ```scala
       val wbuClearMask = Mux(io.wb.RegWrite, sb.mask(io.wb.rd), 0.U)
       ```
3. Call `sb.update` with the generated masks:
   ```scala
   sb.update(isFireSetMask, wbuClearMask)
   ```

---

### Task 5: **Output Connection**
**Objective:** Pass processed signals downstream to the execution unit (EXU) in the `io.to_exu` interface.
**Steps:**
1. Assign signals to `io.to_exu.bits`:
   - Copy all parts of the `DecodeIO` Bundle from `io.from_idu.bits`.
   - For `data` in `DecodeIO`, override specific signals(fuSrc, rfSrc):
     - `fuSrc1` → your processed operand.
     - `fuSrc2` → your selected operand.
2. Ensure `io.to_exu.valid` is set when the instruction is ready to issue.
3. Connect the rest of the bundle to carry forward control flow (`CtrlFlow`) and control signal (`CtrlSignal`).
