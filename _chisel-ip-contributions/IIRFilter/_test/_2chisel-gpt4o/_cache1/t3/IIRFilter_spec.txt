# Design Specification for `IIRFilter` Module

## 1. **Module Name**
   - `IIRFilter`

## 2. **Module Functions**
   The `IIRFilter` module is designed to implement an Infinite Impulse Response (IIR) filter. This module processes input data using a set of numerator and denominator coefficients to produce a filtered output. The primary objective is to perform real-time digital filtering on sequential data streams, which is crucial in signal processing applications like audio filtering, data smoothing, and noise reduction. The module accommodates configurable coefficients to adapt to various filtering requirements.

## 3. **Input and Output Port Descriptions**

### Inputs
1. **clock**: Clock signal (1-bit input).
   - Drives the sequential operations within the module.
2. **reset**: Reset signal (1-bit input).
   - Initializes the internal states and registers to a known state.
3. **io_input_valid**: Input valid signal (1-bit input).
   - Indicates the validity of the incoming data.
4. **io_input_bits**: Input data (4-bit input).
   - Represents the current data sample to be filtered.
5. **io_num_0**: Numerator coefficient 0 (3-bit input).
   - First coefficient for the numerator in the filter equation.
6. **io_num_1**: Numerator coefficient 1 (3-bit input).
   - Second coefficient for the numerator in the filter equation.
7. **io_num_2**: Numerator coefficient 2 (3-bit input).
   - Third coefficient for the numerator in the filter equation.
8. **io_num_3**: Numerator coefficient 3 (3-bit input).
   - Fourth coefficient for the numerator in the filter equation.
9. **io_den_0**: Denominator coefficient 0 (3-bit input).
   - First coefficient for the denominator in the filter equation.
10. **io_den_1**: Denominator coefficient 1 (3-bit input).
    - Second coefficient for the denominator in the filter equation.
11. **io_output_ready**: Output ready signal (1-bit input).
    - Indicates that the downstream module is ready to receive the output data.

### Outputs
1. **io_input_ready**: Input ready signal (1-bit output).
   - Indicates that the module is ready to accept new input data.
2. **io_output_valid**: Output valid signal (1-bit output).
   - Indicates the validity of the output data.
3. **io_output_bits**: Filtered output data (11-bit output).
   - Represents the filtered result of the input data.

## 4. **Internal Working Principle**
The `IIRFilter` module operates as a digital filter, processing input samples through a series of multiply-accumulate operations defined by the given coefficients. The filter uses a combination of numerator and denominator coefficients to recursively compute the output. The internal state machine transitions through various states to manage the input, processing, and output phases, ensuring that each sample is processed correctly according to the IIR filter equation.

## 5. **Implementation Logic Explanation**
The module employs a state machine to manage its operations, with states including input handling, coefficient processing, and output generation. Key operations include:
- **Data Input**: Controlled by `io_input_valid` and `io_input_ready`, data is loaded into an internal register for processing.
- **Coefficient Handling**: The module iterates through the numerator and denominator coefficients, performing multiply-accumulate operations to compute the output.
- **Memory Management**: Input and output samples are stored in internal memories to facilitate recursive filtering operations.
- **Output Generation**: The final filtered output is calculated and presented on `io_output_bits` when `io_output_valid` is asserted.

## 6. **Internally Defined Signal Descriptions**

1. **inputMem**: Input memory (3x4-bit).
   - Stores recent input samples for recursive filtering.
2. **outputMem**: Output memory (2x11-bit).
   - Stores recent output samples for recursive filtering.
3. **coefNum**: Coefficient index (2-bit).
   - Tracks the current coefficient being processed.
4. **state**: State machine register (3-bit).
   - Represents the current state of the module's operation.
5. **inputReg**: Input register (4-bit).
   - Temporarily holds the current input sample.
6. **inputMemAddr**: Input memory address (2-bit).
   - Tracks the current address for input memory operations.
7. **outputMemAddr**: Output memory address (1-bit).
   - Tracks the current address for output memory operations.
8. **REG**: State register (3-bit).
   - Holds the previous state for synchronization purposes.
9. **inputSum**: Input sum register (9-bit).
   - Accumulates the sum of input-related calculations.
10. **outputSum**: Output sum register (15-bit).
    - Accumulates the sum of output-related calculations.
11. **multOutReg**: Multiplier output register (14-bit).
    - Stores the result of the current multiplication operation.
12. **multIn**: Multiplier input (11-bit).
    - Represents the current value to be multiplied by a coefficient.
13. **multCoef**: Multiplier coefficient (3-bit).
    - Represents the current coefficient for multiplication.
14. **multOut**: Multiplier output (14-bit).
    - Result of the multiplication between `multIn` and `multCoef`.

These internal signals facilitate the module's operations, enabling efficient data processing and filtering according to the IIR filter algorithm.
