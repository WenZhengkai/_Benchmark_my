## Module Name
**Allocator**

## Overview
The Allocator module implements a cache way selection mechanism for write operations. It determines which cache way should be written to based on cache hit information and a pseudo-random allocation policy. When a cache hit occurs, it selects the hitting way using priority encoding. When no hit occurs (cache miss), it uses a hash-based pseudo-random algorithm to select a way for allocation.

## Internal Logic

1. **Way Allocation Logic (`alloc_way`):**
   - **For single-way caches (nWays = 1):** Always returns 0 since there's only one way.
   - **For multi-way caches (nWays > 1):** Implements a pseudo-random way selection using XOR-based hashing:

     - Divides `r_metas` into chunks of size `l` bits (where `l = log2Ceil(nWays)`)
     - XORs all chunks together to produce a pseudo-random way index

2. **Write Way Selection:**

3. **Output Assignment:**
   - The selected way index is assigned to `s1_meta_write_way`, indicating which cache way should be updated with new metadata.
