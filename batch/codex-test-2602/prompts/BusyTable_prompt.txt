# Specification

## **Module: BusyTable**

## **Functionality**
The `BusyTable` module is used to track the busy status of registers in a system that supports register renaming and speculative execution. The module receives wake-up signals for registers that have completed execution, rebusy requests for newly allocated registers, and provides a busy table to indicate which registers are currently in use.

The main operations of this module are:
1. **Unbusy** registers that have completed execution and are no longer needed.
2. **Rebusy** registers that are newly allocated or need to be marked busy due to rebusy requests.
3. **Track** speculative execution by masking certain registers and updating the busy table based on the speculative state.

## **Parameters**
- `numPregs` (Int): The number of physical registers being managed by the busy table. This parameter determines the width of the busy table and various input/output vectors.
