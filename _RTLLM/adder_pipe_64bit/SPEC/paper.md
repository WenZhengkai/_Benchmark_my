### 4.1 Network of Task (NoT)

The challenge of processing long texts and performing long-range reasoning is a well-known limitation of large language models (LLMs). To address this issue, we propose the network of task (NoT) framework, which is designed for task decomposition and management. The NoT framework consists of three main components: (1) task builder, an LLM responsible for generating design task documents based on specifications (SPEC); (2) modern HDL gen, an LLM for generating HDL (Hardware Description Language) code; and (3) a task management program, which orchestrates the tasks and ensures the efficient generation and integration of HDL code. These components work together to systematically decompose complex design tasks, manage task execution, and produce correct and complete HDL implementations.

The task builder serves as the starting point of the NoT framework. Its role is to process the given SPEC and generate a design task document. This document outlines the sequence of tasks required to complete the design, with each task corresponding to the implementation of a specific hardware structure described in the SPEC. Each task in the document is detailed with two key components: the “objective,” which specifies the goal of the task, and the “steps,” which describe the procedure for accomplishing the task. Each task description is carefully designed to ensure clarity and consistency, thereby aiding subsequent stages of the NoT framework. By decomposing the overall design into smaller, manageable tasks, the task builder provides a structured foundation for the entire process.

Following the generation of the design task document, the task management program organizes the tasks into groups to facilitate efficient HDL code generation. Tasks are grouped based on a balanced distribution strategy, ensuring that each group contains a manageable number of tasks. For instance, as shown in Figure [x], if the design task document contains six tasks, the program divides them into two groups, each containing three tasks. These groups are processed sequentially by the modern HDL gen component. This grouping strategy ensures that the workload is evenly distributed and manageable, allowing for incremental code generation and verification. By operating on task groups instead of individual tasks, the task management program optimizes the overall process while maintaining flexibility for error handling and refinement.

The modern HDL gen component is responsible for generating Modern HDL code based on the grouped tasks. The process begins with the creation of a code framework, which includes the declaration of libraries, class definitions, port definitions, and other structural elements required by the design. As illustrated in Figure [x], an initial code framework is generated, leaving placeholders for additional content to be filled in during subsequent steps.

Next, the modern HDL gen processes each group of tasks incrementally. For each group, the program provides the LLM with a detailed prompt containing the following information: (1) the SPEC’s basic information, (2) the design task document, (3) the code generated for previously completed tasks, and (4) specific instructions from the task management program regarding the current group of tasks. Based on this input, the modern HDL gen generates the corresponding HDL code for the current task group, while other parts of the code are left as placeholders. Figure [x] provides an example where the output code corresponds to the specified task group, with other sections of the code framework remaining unchanged.

After the code for a task group is generated, it is integrated into the overall code framework. This updated framework is then passed as input for the next group of tasks. This iterative process continues until all tasks have been processed, resulting in a complete HDL implementation. The generated HDL code is subsequently subjected to a linter for syntax verification. If no errors are detected, the code proceeds to the validation phase. If errors are identified, the task management program isolates the problematic code segments, regenerates the corresponding HDL fragments, and reintegrates them into the framework. This process is repeated until all syntax errors are resolved.

In cases where the generated HDL consistently fails validation, the task management program adapts by reducing the number of tasks in each group, thereby increasing the total number of groups. This adjustment allows the modern HDL gen to focus on smaller, more manageable subsets of tasks, improving the likelihood of generating correct code. The iterative refinement process continues until the HDL passes both syntax checks and validation, ensuring the correctness and completeness of the final implementation.

In summary, the NoT framework leverages the strengths of LLMs in task decomposition and code generation while addressing their limitations in long-text processing and reasoning. Through the coordinated efforts of the task builder, task management program, and modern HDL gen, the framework achieves a systematic and efficient approach to HDL design, providing a robust solution for complex hardware development tasks.
