# SPEC

## **Overview**
The `MyFir` class implements a parameterized Finite Impulse Response (FIR) filter in Chisel. The filter's length, bitwidth, and windowing function are configurable, allowing for flexible design and implementation. The FIR filter computes a weighted sum of input samples using a set of coefficients generated by the provided windowing function.

---

## **Class Name**
`MyFir`

---

## **Parameters**
The `MyFir` class takes the following parameters:

1. **`length`**: (Int)  
   The number of taps (coefficients) in the FIR filter. This determines the size of the delay line and the number of coefficients used in the filter.

2. **`bitwidth`**: (Int)  
   The bitwidth of the input data and the coefficients. It determines the precision of the calculations and the size of the input/output signals.

3. **`window`**: (Int, Int) => Seq[Int]  
   A function that generates the filter coefficients based on the specified `length` and `bitwidth`. The function returns a sequence of integers representing the scaled coefficients.

---

## **Input/Output Interface**

### **Inputs**
- **`in`**: `UInt` (`bitwidth.W`)  
  The input signal to the FIR filter. It is a single unsigned integer value representing the current sample.

### **Outputs**
- **`out`**: `UInt` (`(bitwidth * 2 + length - 1).W`)  
  The filtered output signal of the FIR filter. The bitwidth of the output accommodates the potential bit growth caused by multiplication and addition during the filtering process.

---

## **Internal Logic**

1. **Coefficient Calculation:**  
   - The filter coefficients are generated using the provided `window` function.
   - The coefficients are converted to unsigned integers.

2. **Delay Line Creation:**  
   - A sequence of delay registers is created to store the past `length` input samples.
   - The delay chain is implemented shift the input signal through the registers.

3. **Multiplication:**  
   - Each delayed sample is multiplied by its corresponding coefficient.
   - The multiplication results are stored in a sequence called `mults`.

4. **Summation:**  
   - All the multiplier outputs are summed together to compute the final filter output.
   - The width of the summation result accommodates bit growth resulting from multiple additions.

5. **Output Connection:**  
   - The computed sum (filtered signal) is assigned to the `io.out` port.
